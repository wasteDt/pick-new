# Pick Quote 项目实现说明

## 1. 项目定位

`Pick Quote` 是一个基于 `Plasmo + React + TypeScript` 实现的浏览器扩展，用来在浏览网页时快速保存文本、图片、链接和页面截图，并在扩展管理页中进行查看、搜索和导出。

这个项目当前采用“本地优先”的实现方式：

- 默认把数据保存到浏览器本地 `IndexedDB`
- 可选接入 `Convex` 做多设备同步
- 管理界面运行在扩展的 `options.html` 页面中

## 2. 技术栈

- 扩展框架：`Plasmo`
- 前端：`React 18`
- 语言：`TypeScript`
- UI 组件：`MUI`
- 本地存储：`IndexedDB`
- 云同步：`Convex`
- ZIP 导出：`JSZip`
- 图片导出：`html2canvas`

实现位置：

- 项目配置与脚本：`pick-quote/package.json`
- 扩展权限配置：`pick-quote/package.json` 中的 `manifest`

## 3. 整体架构

项目可以分成 5 层：

1. 页面采集层  
   负责从网页里提取选中文本、来源页面信息、截图等内容。

2. 扩展后台层  
   负责处理右键菜单、接收内容脚本消息、统一生成条目并入库。

3. 数据仓库层  
   负责封装“保存、删除、搜索、同步”等业务流程。

4. 本地存储层  
   负责 `IndexedDB` 的表结构、索引和 CRUD。

5. 管理与导出层  
   负责展示条目、详情查看、导出 ZIP、导出分享图片。

## 4. 核心数据模型

项目的核心数据结构是 `Item`，表示一条收藏内容。

主要字段包括：

- `id`：本地主键
- `type`：内容类型，支持 `text`、`image`、`link`、`snapshot`
- `content`：正文、图片地址、链接地址或截图内容
- `context`：选中文本所在段落的上下文
- `source`：来源标题、URL、站点信息
- `createdAt` / `updatedAt`：创建和更新时间
- `hash`：用于去重
- `syncStatus`：同步状态
- `remoteId`：云端记录 id

实现位置：

- 类型定义：`pick-quote/src/types/index.ts`

## 5. 页面采集是如何实现的

### 5.1 右键菜单采集

扩展安装时会注册四类右键菜单：

- 选中文本
- 图片
- 链接
- 页面可视区截图

用户点击菜单后，后台脚本会根据菜单类型构造 `Item`，然后调用统一保存入口。

实现位置：

- 右键菜单注册：`pick-quote/src/background.ts`
- 右键菜单点击处理：`pick-quote/src/background.ts`
- 保存入口：`pick-quote/src/repository/items.ts` 中的 `saveCapturedItem`

### 5.2 内容脚本采集

内容脚本注入到所有网页中，提供两部分能力：

- 监听 `Ctrl + Shift + S` 快捷键
- 提取选中文本的上下文和定位信息

提取逻辑包括：

- `deriveContext()`：提取选中文本前后各一小段内容，以及所在段落全文
- `deriveAnchor()`：为当前选区生成一个 CSS 选择器
- `getCssSelector()`：向上遍历 DOM，拼出一个可用于回溯定位的选择器路径

采集完成后，内容脚本通过 `chrome.runtime.sendMessage` 发送给后台脚本统一处理。

实现位置：

- 内容脚本入口：`pick-quote/src/content-scripts/capture.ts`
- 快捷键监听：`pick-quote/src/content-scripts/capture.ts`
- 上下文提取：`pick-quote/src/content-scripts/capture.ts`
- 锚点定位生成：`pick-quote/src/content-scripts/capture.ts`

## 6. 后台层是如何工作的

后台脚本是扩展的数据入口中枢，主要负责三件事：

### 6.1 注册右键菜单

扩展安装后通过 `chrome.runtime.onInstalled` 创建右键菜单。

实现位置：

- `pick-quote/src/background.ts`

### 6.2 接收采集结果并转成本地条目

后台层会同时处理两类来源：

- 右键菜单直接生成的数据
- 内容脚本通过消息发送的数据

对于截图类内容，后台调用 `chrome.tabs.captureVisibleTab` 生成 `data:image/png` 形式的截图数据，再写入条目。

实现位置：

- 菜单点击处理：`pick-quote/src/background.ts`
- 页面截图：`pick-quote/src/background.ts`
- 内容脚本消息接收：`pick-quote/src/background.ts`

### 6.3 打开管理页

点击扩展图标后，后台脚本直接打开 `options.html`，因此这个项目当前没有单独的 popup 页面，主界面就是管理页。

实现位置：

- `chrome.action.onClicked`：`pick-quote/src/background.ts`

## 7. 本地存储是如何实现的

### 7.1 为什么使用 IndexedDB

本地存储采用浏览器原生 `IndexedDB`，适合存放截图、文本和结构化元数据，不依赖外部服务，也符合扩展“离线可用”的需求。

### 7.2 数据库结构

数据库名为 `pickquote-db`，当前版本号是 `3`。

包含的对象仓库：

- `items`
- `categories`
- `sources`

其中 `items` 是最核心的表。

实现位置：

- 数据库初始化：`pick-quote/src/database/index.ts`

### 7.3 items 表索引设计

`items` 表建立了这些索引：

- `type`
- `createdAt`
- `updatedAt`
- `sourceSite`
- `categoryId`
- `hash`
- `syncStatus`
- `remoteId`

这些索引分别用于：

- 列表排序
- 搜索过滤
- 去重
- 同步状态查询
- 云端记录反查

实现位置：

- 索引创建：`pick-quote/src/database/index.ts`

### 7.4 本地 CRUD 封装

本地数据库层封装了完整的增删查改：

- `addItem`
- `getRecent`
- `searchItems`
- `updateItem`
- `deleteItem`
- `exportItems`
- `getItemByRemoteId`
- `getItemsBySyncStatus`
- `upsertRemoteItem`

实现位置：

- `pick-quote/src/database/index.ts`

## 8. 去重和数据标准化是如何实现的

### 8.1 哈希去重

项目通过 `url + content` 计算 `SHA-256`，把结果作为 `hash` 保存。  
如果本地已存在相同 `hash` 的条目，则跳过写入。

实现位置：

- 哈希计算：`pick-quote/src/utils/index.ts` 中的 `sha256`、`computeItemHash`
- 去重逻辑：`pick-quote/src/database/index.ts` 中的 `addItem`

### 8.2 标准化字段

保存前会补齐这些信息：

- `updatedAt`
- `sourceSite`
- `hash`
- `syncStatus`

这样业务层和 UI 层就不用每次手动判断。

实现位置：

- 标准化逻辑：`pick-quote/src/database/index.ts` 中的 `addItem`

## 9. 业务仓库层是如何实现的

`repository/items.ts` 是本项目的业务中心，连接了：

- 采集入口
- 本地数据库
- 云同步逻辑
- 删除逻辑
- 后台同步调度

### 9.1 保存条目

`saveCapturedItem` 的处理流程是：

1. 获取同步配置和当前设备 id
2. 生成创建时间
3. 写入本地数据库
4. 标记同步状态为 `pending`
5. 异步触发云端同步

实现位置：

- `pick-quote/src/repository/items.ts` 中的 `saveCapturedItem`

### 9.2 删除条目

删除时会先删本地。  
如果该条目已经同步到云端，并且开启了云同步，则继续向 Convex 发起远端删除。

实现位置：

- `pick-quote/src/repository/items.ts` 中的 `deleteItem`

### 9.3 后台同步调度

仓库层通过两个 Promise 锁避免重复并发执行：

- `pushSyncPromise`
- `pullSyncPromise`

实现位置：

- `pick-quote/src/repository/items.ts`

## 10. 云同步是如何实现的

这个项目的同步模式是“本地优先 + 可选云同步”。

### 10.1 同步配置

同步配置来源包括：

- 构建时环境变量
- `chrome.storage.local`

关键配置项：

- `PLASMO_PUBLIC_CONVEX_URL`
- `PLASMO_PUBLIC_PICKQUOTE_USER_ID`

本地还会保存：

- `deviceId`
- `lastPulledAt`

实现位置：

- 同步配置读取：`pick-quote/src/sync/config.ts`

### 10.2 哪些内容会同步

当前不是所有条目都会上传。

被排除的内容有：

- `snapshot`
- `image`
- `data:image...` 形式的大图片内容

也就是说，当前更偏向同步文本和链接，避免把大体积图片直接推到云端。

实现位置：

- 同步过滤逻辑：`pick-quote/src/repository/items.ts` 中的 `canSyncItem`

### 10.3 推送到云端

待同步条目会被批量读取，然后逐条调用 `pushItemToCloud` 上传到 Convex。  
成功后回写：

- `syncStatus = "synced"`
- `remoteId`
- `lastSyncedAt`

失败则标记为 `failed`。

实现位置：

- 推送调度：`pick-quote/src/repository/items.ts` 中的 `syncPendingItems`
- Convex 客户端调用：`pick-quote/src/sync/convex.ts` 中的 `pushItemToCloud`

### 10.4 从云端拉取

拉取时会根据 `lastPulledAt` 调用 Convex 查询增量数据，然后逐条合并到本地库中。

实现位置：

- 拉取调度：`pick-quote/src/repository/items.ts` 中的 `syncFromCloud`
- Convex 客户端调用：`pick-quote/src/sync/convex.ts` 中的 `pullItemsSince`
- 本地合并逻辑：`pick-quote/src/database/index.ts` 中的 `upsertRemoteItem`

### 10.5 管理页打开时触发同步

当前同步不是实时订阅，而是在管理页加载时主动执行一次：

1. 先从云端拉取
2. 再把本地待同步条目推送到云端

实现位置：

- 页面触发点：`pick-quote/src/options.tsx`
- 执行入口：`pick-quote/src/repository/items.ts` 中的 `runBackgroundSync`

## 11. Convex 服务端是如何实现的

### 11.1 云端表结构

Convex 目前只定义了一张 `items` 表，用于保存云端同步条目。

字段包括：

- `userId`
- `type`
- `content`
- `source`
- `sourceSite`
- `createdAt`
- `updatedAt`
- `categoryId`
- `note`
- `hash`
- `deviceId`

并建立了这些索引：

- `by_user_createdAt`
- `by_user_type`
- `by_user_hash`
- `by_user_site`

实现位置：

- `pick-quote/convex/schema.ts`

### 11.2 云端接口

Convex 目前提供 3 个主要接口：

- `upsertFromSync`：上传时按 `remoteId` 或 `hash` 做幂等更新
- `listSince`：按用户拉取某个时间点之后更新过的条目
- `removeById`：删除远端条目

实现位置：

- `pick-quote/convex/items.ts`

## 12. 管理页是如何实现的

### 12.1 页面结构

管理页使用 `React + MUI` 实现，包含：

- 顶部标题区域
- 搜索框
- 类型筛选
- 导出按钮
- 瀑布流式条目列表
- 条目详情弹窗

实现位置：

- 页面入口：`pick-quote/src/options.tsx`

### 12.2 搜索与过滤

管理页会把输入的关键字和类型条件传给仓库层，再由本地数据库完成筛选。

实现位置：

- 页面筛选逻辑：`pick-quote/src/options.tsx`
- 搜索实现：`pick-quote/src/repository/items.ts` 中的 `searchItems`
- 数据库过滤：`pick-quote/src/database/index.ts` 中的 `searchItems`

### 12.3 条目卡片

列表项卡片支持：

- 显示不同类型内容
- 打开来源页面
- 删除
- 导出为图片

实现位置：

- 卡片组件：`pick-quote/src/components/ItemCard.tsx`

### 12.4 条目详情弹窗

点击卡片后，会打开一个详情弹窗，支持：

- 查看完整内容
- 查看上下文段落
- 打开来源
- 导出图片

实现位置：

- 详情弹窗：`pick-quote/src/components/ItemDialog.tsx`

## 13. 导出功能是如何实现的

### 13.1 ZIP 导出

导出按钮会读取所有条目并调用 `toZip`。

导出策略如下：

- 文本和链接写入 `export.md`
- 图片和截图如果是 `data:image`，则保存到 `images/` 目录
- Markdown 中插入图片引用路径

实现位置：

- 页面导出入口：`pick-quote/src/options.tsx`
- ZIP 生成：`pick-quote/src/export/zipExport.ts`

### 13.2 图片导出

条目卡片和详情弹窗都支持导出分享图。

实现方式：

1. 用隐藏的 `ShareCard` 组件生成分享卡片 DOM
2. 使用 `html2canvas` 把 DOM 转成 canvas
3. 再导出成 PNG

实现位置：

- 图片导出函数：`pick-quote/src/export/imageExport.ts`
- 卡片导出入口：`pick-quote/src/components/ItemCard.tsx`
- 详情导出入口：`pick-quote/src/components/ItemDialog.tsx`

## 14. UI 主题是如何实现的

项目会根据系统深浅色偏好自动切换主题，并通过 MUI 主题系统统一管理颜色和样式。

实现位置：

- 主题创建：`pick-quote/src/theme/index.ts`
- 页面使用：`pick-quote/src/options.tsx`

## 15. 当前实现特点与限制

### 优点

- 结构清晰，分层明确
- 本地优先，离线可用
- 右键菜单、快捷键、截图、导出都已具备
- 云同步是可选能力，不依赖时也能独立使用

### 当前限制

- 项目当前没有独立 popup，点击图标直接进入管理页
- 云同步没有完整认证体系，`userId` 主要来自配置
- 图片和截图不会完整参与多端同步
- `Convex` 的增量查询目前仍有进一步优化空间
- 仓库中部分旧中文文案存在编码问题，但不影响整体业务结构理解
